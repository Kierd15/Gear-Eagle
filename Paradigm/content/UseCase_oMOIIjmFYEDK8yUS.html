Description

El sistema debe implementar un "borrado lógico" (cambio de estado a Inactivo) para impedir el acceso a ex-empleados, manteniendo la integridad de los registros que estos crearon en el pasado.

Scenario
Steps
1. El ADMINISTRADOR selecciona la opción “Dar de Baja Usuario”.
2. El sistema muestra el listado de usuarios activos.
3. El ADMINISTRADOR selecciona el usuario que desea dar de baja.
4. El sistema muestra la información del usuario seleccionado (Nombre, ID, Rol, Estado).
5. El ADMINISTRADOR confirma la acción de baja.
6. El sistema solicita confirmación adicional de seguridad (ejemplo: confirmación de contraseña del Administrador).
7. El ADMINISTRADOR confirma la operación.
8. if la verificación de seguridad es correcta
    8.1. AND el usuario seleccionado se encuentra en estado Activo) THEN
    8.2. • El sistema cambia el estado del usuario a “Inactivo”.
    8.3. • El sistema bloquea el acceso del usuario al sistema.
    8.4. • El sistema mantiene intactos los registros históricos creados por el usuario.
    8.5. • El sistema muestra mensaje: "Usuario desactivado correctamente".
9. else
    9.1. • Mostrar mensaje: "No fue posible realizar la operación".
   end if
10. Fin del caso de uso.
