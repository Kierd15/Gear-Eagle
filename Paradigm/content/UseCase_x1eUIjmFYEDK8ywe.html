Use Case Note
■ Workflow
    • // Write down briefly how user perform the work
    • • El almacenista recibe los productos junto con la factura u orden de soporte.
    • • El almacenista ingresa al sistema y selecciona la opción "Crear ingreso".
    • • El sistema muestra el formulario de registro de ingreso.
    • • El almacenista ingresa la fecha, el responsable y el número del documento soporte.
    • • El almacenista agrega uno o varios productos indicando la cantidad correspondiente.
    • • El almacenista confirma el registro del ingreso.
    • • El sistema valida la información ingresada.
    • • Si la información es correcta, el sistema guarda el ingreso y actualiza el inventario.
    • • El sistema muestra un mensaje de confirmación.
■ Business Logic
    • // Write down what user expect the system to react upon certain condition (e.g. low inventory alert level)
    • • La fecha del ingreso no puede ser futura.
    • • El número de documento soporte es obligatorio.
    • • El número de documento no puede estar previamente registrado en el sistema.
    • • El ingreso debe contener al menos un producto.
    • • Cada producto debe estar previamente registrado en el sistema.
    • • La cantidad ingresada debe ser mayor a cero.
    • • Solo el usuario con rol de almacenista puede registrar ingresos.
    • • Al confirmar el ingreso, el sistema debe actualizar automáticamente el stock.
    • • El sistema debe registrar quién realizó el ingreso y la fecha del registro.

Scenario
Steps
1. El ALMACENISTA selecciona la opción “Crear ingreso” en el sistema.
2. El sistema muestra el formulario de registro de ingreso.
3. El ALMACENISTA ingresa la fecha del ingreso.
4. El ALMACENISTA ingresa el número de documento soporte (factura u orden).
5. El ALMACENISTA ingresa su identificación como responsable del registro.
    5.1. El ALMACENISTA agrega productos al ingreso.
6. while WHILE (desee agregar productos)
    6.1. • Selecciona un producto del sistema.
    6.2. • Ingresa la cantidad correspondiente.
    6.3. • El sistema valida que la cantidad sea mayor a cero.
    6.4. • El sistema agrega el producto a la lista del ingreso.
   end while
7. El almacenista confirma el registro del ingreso.
8. El sistema valida la información ingresada:
9. if IF (el documento ya existe) THEN
    9.1. • Mostrar mensaje: "Documento duplicado".
    9.2. • Permitir corrección.
10. else if ELSE IF (no hay productos agregados) THEN
    10.1. • Mostrar mensaje: "Debe agregar al menos un producto".
11. else
    11.1. • Guardar el ingreso.
    11.2. • Actualizar el stock de cada producto.
    11.3. • Registrar fecha y usuario que realizó la operación.
    11.4. • Mostrar mensaje: "Ingreso registrado correctamente".
    end if
12. Fin del caso de uso.
