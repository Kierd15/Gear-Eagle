  SISTEMA : System

Módulo de Usuarios (Gestión de Personal y Acceso)

Este módulo controla quién opera el sistema y qué permisos tiene.

Módulo de Productos (Catálogo e Inventario Maestro)

El catálogo central de todo lo que el taller puede vender o usar.

Módulo de Automotores (Gestión de Vehículos)

Base de datos de las motos vinculadas a sus propietarios.

Módulo de Servicios y Orden de Trabajo

Donde el mecánico realiza el diagnóstico y solicita los materiales.

 Módulo de Ingresos (Compras y Abastecimiento)

Proceso de entrada de mercancía para alimentar el stock.

Módulo de Autorizaciones (Portal del Cliente)

El puente de comunicación y aprobación para el presupuesto.

Módulo de Recibos y Facturación

Cierre de la orden, liquidación y descarga final de inventario.

  REGISTRO DE PRODUCTOS : Use Case

El sistema debe permitir registrar nuevos repuestos en el catálogo incluyendo: Código/SKU, Nombre, Marca, Categoría, Precio de Venta, Cantidades recibidas, Sugerido y Punto de Reorden (Stock mínimo). 

 

  BUSCADOR DE PRODUCTO : Use Case

El sistema debe ofrecer una búsqueda avanzada por nombre, código o compatibilidad de marca para agilizar el trabajo del mecánico y almacenista.

  MODIFICAR PRODUCTO : Use Case

El sistema debe permitir la actualización masiva o individual de los precios de venta de los productos del catálogo.

  DESACTIVAR PRODUCTOS : Use Case

El sistema debe permitir deshabilitar productos que ya no se comercializan o están obsoletos para evitar su selección en nuevas órdenes. 

 

  VERIFICACION DE PAGO : Use Case

El sistema debe verificar que la factura haya sido pagada correctamente antes de cerrar la orden de servicio y habilitar beneficios asociados como puntos de fidelización.

 

  REGISTRO DE PERSONAL : Use Case

El sistema debe permitir al Administrador crear perfiles de usuario capturando: Nombre, ID, Teléfono, Correo electrónico, Contraseña y Asignación de Rol (Administrador, Almacenista, Personal Operativo).

  CONSULTA DE USUARIOS : Use Case

El sistema debe permitir listar y filtrar a los empleados por su rol o estado de disponibilidad.

  GESTION DE PERFIL : Use Case

El sistema debe permitir actualizar los datos de contacto y el restablecimiento de contraseñas de manera segura

  BAJA DE USUARIOS : Use Case

El sistema debe implementar un "borrado lógico" (cambio de estado a Inactivo) para impedir el acceso a ex-empleados, manteniendo la integridad de los registros que estos crearon en el pasado. 

 

  AUTENTICACION : Use Case

El sistema debe validar las credenciales de acceso y restringir la visibilidad de los módulos según el nivel de permiso del rol. 

 

  REGISTRO DE MOTO : Use Case

El sistema debe permitir crear la ficha técnica de la moto: Placa (Obligatorio/Único), Marca, Modelo/Año, Color, Número de Motor y Kilometraje actual. 

 

  ACTUALIZACION DE DATOS : Use Case

El sistema debe permitir corregir o actualizar la información técnica del vehículo o realizar el traspaso de dueño en el sistema.

  VINCULACION DE CLIENTE : Use Case

El sistema debe permitir asociar cada moto a los datos de un cliente (Nombre, Cédula, Teléfono) para trazabilidad. 

 

  HOJA DE VIDA : Use Case

El sistema debe permitir visualizar el historial completo de todos los servicios, repuestos instalados y diagnósticos previos realizados a una moto mediante la búsqueda por placa. 

 

  SEGUIMIENTO DE ESTADOS : Use Case

El sistema debe actualizar el estado de la orden (Recibida -> En Diagnóstico -> Pendiente Autorización -> En Reparación -> Lista para Entrega).

  REGISTRO DE DIAGNOSTICO : Use Case

El sistema debe permitir al Mecánico redactar un diagnóstico técnico detallado y adjuntar el estado de la moto.

  ASIGNACION DE MANO DE OBRA : Use Case

El sistema debe permitir agregar los servicios realizados (mano de obra) con sus respectivos tiempos y costos

  SOLICITUD DE REPUESTOS : Use Case

El sistema debe permitir al Mecánico agregar a la orden los repuestos necesarios tomados del Módulo de Productos, los cuales quedarán en estado "Pendiente de Autorización

  APERTURA DE ORDEN : Use Case

El sistema debe permitir generar una Orden de Servicio vinculada a una Placa, registrando el estado en que se recibe la moto y el motivo de ingreso. 

 

  CREAR INGRESOS : Use Case

Asegurar el registro y control adecuado de los nuevos repuestos que ingresan al almacén, garantizando trazabilidad, actualización del inventario y respaldo documental.

  CONSULTAR COMPRAS : Use Case

 El sistema debe permitir consultar cuándo y a qué precio se compraron determinados repuestos para analizar variaciones de costo. 

 

  MODIFICACIÓN DE INGRESO : Use Case

El sistema debe permitir editar un ingreso solo si se detectan errores en la cantidad reportada antes de que el producto sea utilizado.

  CONSULTA POR PLACA : Use Case

El sistema debe habilitar una interfaz web pública donde el Cliente, ingresando únicamente su Placa, pueda ver el estado de su moto. 

 

  VISUALIZACION DE PRESUPUESTO : Use Case

El cliente debe poder leer el detalle de los repuestos y servicios sugeridos por el mecánico con sus precios unitarios y totales. 

 

  APROBACIÓN : Use Case

El sistema debe permitir al cliente autorizar o rechazar cada ítem del presupuesto de forma individual o global. 

 

  NOTIFICACIÓN : Use Case

Permite enviar alertas automáticas a clientes y administradores según eventos relevantes en la plataforma.

 

  GENERACIÓN DE RECIBO : Use Case

El sistema debe generar automáticamente un documento de cobro que incluya únicamente los servicios y repuestos autorizados por el cliente en el módulo anterior. 

 

  REPORTE DE VENTAS : Use Case

 El sistema debe permitir al Administrador consultar y reimprimir recibos históricos para auditoría y contabilidad. 

 

  ANULACIÓN DE FACTURA : Use Case

El sistema debe permitir anular un recibo por error administrativo, lo cual debe reintegrar los repuestos al stock y reabrir la orden de servicio. 

 

  REGISTRO DE CLIENTE : Use Case

Permite registrar nuevos clientes en la plataforma, almacenando información detallada que facilitará futuras ventas y comunicación.

  PROGRAMA DE FIDELIZACION : Use Case

El sistema debe permitir acumular puntos o beneficios de fidelización para los clientes por cada orden de servicio pagada, los cuales podrán consultarse en el portal y utilizarse en futuros servicios.

  CALIFICACION DEL SERVICIO : Use Case

El sistema debe permitir que el cliente califique el servicio recibido mediante una puntuación y un comentario opcional una vez finalizada la orden de servicio. La calificación debe quedar registrada para consulta del administrador.
