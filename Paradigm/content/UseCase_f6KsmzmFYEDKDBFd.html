Use Case Note
■ Workflow
    • // Write down briefly how user perform the work
    • El cliente realiza un servicio en el taller.
    • El sistema genera la orden de servicio.
    • El cliente paga la factura del servicio.
    • El sistema detecta que la orden fue pagada correctamente.
    • El sistema calcula los puntos de fidelización generados según el valor del servicio.
    • El sistema registra los puntos en la cuenta del cliente.
    • El cliente puede ingresar al portal del cliente.
    • El cliente consulta su saldo de puntos acumulados.
    • El cliente puede usar los puntos como descuento o beneficio en futuros servicios.
■ Business Logic
    • // Write down what user expect the system to react upon certain condition (e.g. low inventory alert level)
    • Los puntos de fidelización solo se generan cuando la orden de servicio ha sido pagada completamente.
    • Los puntos se calculan según el valor total del servicio realizado.
    • Cada cliente tiene una cuenta de puntos asociada a su perfil.
    • Los puntos acumulados pueden ser consultados en el portal del cliente.
    • Los puntos pueden ser canjeados en servicios futuros como descuentos o beneficios.
    • El sistema debe registrar cada movimiento de puntos (acumulación o uso).
    • Solo se pueden usar puntos si el cliente tiene saldo suficiente disponible.
    • El administrador puede consultar el historial de puntos de cada cliente.

Scenario
Steps
1. El CLIENTE solicita un servicio en el taller.
2. El sistema genera una orden de servicio.
3. El servicio es realizado por el personal del taller.
4. El sistema genera la factura correspondiente al servicio.
5. El CLIENTE realiza el pago del servicio.
6. El sistema confirma que la factura fue pagada correctamente.
7. El sistema calcula los puntos de fidelización generados según el valor del servicio.
8. El sistema registra los puntos en la cuenta del cliente.
9. El CLIENTE accede al portal del cliente.
10. El CLIENTE consulta su saldo de puntos acumulados.
11. El CLIENTE puede utilizar los puntos en futuros servicios como beneficio o descuento.
12. El caso de uso finaliza.
