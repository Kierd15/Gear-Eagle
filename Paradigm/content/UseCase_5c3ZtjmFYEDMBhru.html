Use Case Note
■ Workflow
    • // Write down briefly how user perform the work
    • El usuario selecciona la opción “Registrar Cliente”.
    • El sistema muestra el formulario de registro.
    • El usuario ingresa los datos obligatorios (nombre, documento de identidad, teléfono, correo electrónico, dirección, etc.).
    • El usuario confirma la información y envía el formulario.
    • El sistema valida los datos ingresados.
    • Si la validación es correcta, el sistema registra al cliente y genera un ID único.
    • El sistema muestra un mensaje de confirmación de registro exitoso.
    • Opcionalmente, el sistema envía una notificación o correo de bienvenida al cliente.
■ Business Logic
    • // Write down what user expect the system to react upon certain condition (e.g. low inventory alert level)
    • El sistema debe validar que todos los campos obligatorios estén completos antes de permitir el registro.
    • El número de documento y/o correo electrónico no deben estar previamente registrados en el sistema (evitar duplicados).
    • El formato del correo electrónico y teléfono debe validarse automáticamente.
    • El sistema debe generar un identificador único para cada cliente registrado.
    • Los datos deben almacenarse de forma segura cumpliendo con políticas de protección de datos.
    • Si existe un intento de registro con datos ya existentes, el sistema debe notificarlo y ofrecer la opción de actualizar información.
    • El sistema debe registrar fecha, hora y usuario que realizó el registro para fines de auditoría.
    • Solo usuarios con permisos autorizados pueden crear o modificar registros de clientes.

Scenario
Steps
1. Usuario CLIENTE o MECANICO accede al apartado de registro.
2. Sistema muestra formulario con campos obligatorios y opcionales:
    2.1. Nombre, apellido
    2.2. DNI o identificación
    2.3. Fecha de nacimiento
    2.4. Dirección completa
    2.5. Teléfono y correo electrónico
    2.6. Información de motos (marca, modelo, año, historial de servicios)
    2.7. Preferencias de contacto y servicios futuros
3. Usuario completa los datos.
4. Sistema valida cada campo:
5. if algún dato obligatorio está vacío
    5.1. muestra mensaje de error y solicita completar.
   end if
6. if formato de correo o teléfono incorrecto
    6.1. muestra mensaje de corrección.
   end if
7. if DNI ya existe en base de datos
    7.1. alerta de cliente ya registrado.
   end if
8. Usuario corrige errores y vuelve a enviar.
9. Sistema guarda los datos en la base de datos.
10. Sistema confirma registro con mensaje visual y envío de correo de bienvenida opcional.
11. Sistema genera un ID único de cliente y registra la fecha de creación.
12. ADMINISTRADOR puede consultar el perfil del cliente recién registrado.
13. Flujo alternativo: si el sistema detecta un error de conexión o de base de datos muestra mensaje de fallo y permite reintentar.
