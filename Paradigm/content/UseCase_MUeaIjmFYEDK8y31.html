Use Case Note
■ Workflow
    • // Write down briefly how user perform the work
    • El cliente accede al portal o aplicación y visualiza los presupuestos o servicios pendientes de aprobación.
    • Selecciona el ítem a aprobar y revisa su detalle completo.
    • Confirma la aprobación mediante un mecanismo digital (firma biométrica, PIN, OTP, etc.).
    • El sistema valida la autenticidad del consentimiento.
    • La aprobación queda registrada con fecha y usuario.
    • Se notifica automáticamente al área operativa para iniciar la obra.
    • Se muestra comprobante de aprobación al cliente.
■ Business Logic
    • // Write down what user expect the system to react upon certain condition (e.g. low inventory alert level)
    • La aprobación digital debe tener validez legal y cumplir con normativas vigentes (ej. firma electrónica reconocida).
    • Solo usuarios autenticados y autorizados pueden aprobar.
    • El sistema debe validar que el presupuesto o servicio esté en estado pendiente.
    • La firma digital o consentimiento debe ser verificada antes de registrar la aprobación.
    • La aprobación debe quedar almacenada de forma segura y auditable.
    • El sistema debe notificar a los departamentos correspondientes para la ejecución inmediata.
    • En caso de rechazo, debe quedar registro y notificación para gestión.
    • El sistema debe manejar fallos en la firma con reintentos y mensajes claros al usuario.

Scenario
Steps
1. El CLIENTE accede al portal web o aplicación.
2. El sistema muestra los presupuestos o servicios pendientes de aprobación.
3. ElCLIENTE selecciona el presupuesto o servicio a aprobar.
4. El sistema muestra el detalle completo del presupuesto o servicio.
5. El CLIENTE revisa la información.
6. El CLIENTE acepta y da su consentimiento para aprobar digitalmente.
7. El sistema despliega el mecanismo de firma digital o consentimiento electrónico.
8. El CLIENTE firma o confirma su aprobación mediante el mecanismo habilitado (firma biométrica, PIN, OTP, etc.).
9. El sistema valida la autenticidad de la firma o consentimiento.
10. El sistema registra la aprobación con fecha, hora, datos del cliente y detalle de la aprobación.
11. El sistema notifica automáticamente al área operativa para iniciar la obra o servicio.
12. El sistema muestra al cliente un comprobante de aprobación (pantalla o descarga PDF).
13. El caso de uso finaliza.
