Use Case Note
■ Workflow
    • // Write down briefly how user perform the work
    • El cliente accede al portal web o aplicación.
    • Selecciona la opción Visualización de Presupuesto.
    • Ingresa el número de placa o código de presupuesto.
    • El sistema valida la información ingresada.
    • El sistema consulta la base de datos.
    • El sistema muestra el detalle completo del presupuesto.
    • El cliente puede descargar el presupuesto en PDF o regresar.
■ Business Logic
    • // Write down what user expect the system to react upon certain condition (e.g. low inventory alert level)
    • El presupuesto debe estar previamente registrado y asociado a un vehículo.
    • Solo se mostrará información correspondiente al presupuesto consultado.
    • El sistema debe desglosar claramente:
    • Mano de obra
    • Repuestos
    • Cantidades
    • Precios unitarios
    • Subtotales
    • Impuestos
    • Total final
    • El presupuesto debe indicar su estado (pendiente, aprobado, rechazado, vencido).
    • La consulta es solo de lectura y no permite modificaciones.
    • El sistema debe garantizar que los cálculos del total coincidan con el desglose.
    • El sistema puede registrar la visualización para fines de seguimiento.

Scenario
Steps
1. CLIENTE accede al portal web o aplicación del sistema.
2. El sistema muestra la opción Visualización de Presupuesto.
3. El CLIENTE selecciona la opción.
4. El sistema despliega un formulario para ingresar:
    4.1. Número de placa o
    4.2. Código único del presupuesto.
5. El CLIENTE ingresa el dato correspondiente.
6. El CLIENTE presiona el botón Consultar.
7. El sistema valida que el dato ingresado no esté vacío y cumpla el formato esperado.
8. El sistema consulta la base de datos con el dato proporcionado.
9. El sistema verifica si existe un presupuesto asociado y válido.
10. Si existe, el sistema muestra:
    10.1. Detalle de mano de obra (descripción, cantidad, precio unitario, subtotal)
    10.2. Detalle de repuestos (descripción, cantidad, precio unitario, subtotal)
    10.3. Subtotales y totales
    10.4. Impuestos aplicados
    10.5. Estado del presupuesto (pendiente, aprobado, rechazado, vencido)
11. El CLIENTE puede:
    11.1. Descargar el presupuesto en formato PDF.
    11.2. Regresar para realizar otra consulta o salir.
12. El caso de uso finaliza.
